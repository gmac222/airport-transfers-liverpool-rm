# RM Driver — native app

Mobile native shell (iOS + Android) for the RM Transfers driver portal,
built with Vite + React + Capacitor. Hits the same `/api/*` endpoints
deployed at https://airport-transfers-liverpool-rm.vercel.app.

## Local dev (browser)

```bash
npm install
npm run dev
```

Opens at http://localhost:5173. Login uses the live `/api/login` endpoint.

## Native build flow

1. `npm run build` — produces `dist/`
2. `npx cap sync` — copies `dist/` into the iOS + Android projects
3. Open the native project to build/run on a device:
   - **Android:** `npx cap open android` (requires Android Studio)
   - **iOS:** `npx cap open ios` (requires a Mac with Xcode)

Bundle ID: `uk.co.rmtransfers.driverapp` · Display name: **RM Driver**.

## Push notifications (next steps)

`src/push.js` registers a device token but currently just logs it. To
actually deliver pushes you need:

1. **Android (FCM):** create a Firebase project, download
   `google-services.json` into `android/app/`, configure FCM in your backend.
2. **iOS (APNs):** Apple Developer account, push capability + APNs key,
   wire to your backend (or use Firebase as a unified gateway).
3. **Backend:** when a job is reassigned, look up the driver's stored token
   in Airtable and send via FCM/APNs in addition to the existing SMS.

## Env

`VITE_API_BASE` overrides the API host at build time. Default points at
the production Vercel deployment so the same artifact runs on iOS, Android,
and `npm run dev`.
